## 题目网页的交互功能

打开任意 `第NN套/题目.html`：

1. **点击作答** —— 单选 / 判断题的每个选项都可点击选中（单选行为）。
2. **一键判卷** —— 右上角红色【📝 判卷】按钮，按客观题（单选+判断，每题 2 分）计分，得分用大红字显示。
3. **未做提醒** —— 若有客观题未作答，判卷时弹窗列出题号，只对已作答的题目计分。
4. **错题处理** —— 判错的题整块高亮（强红反差色），题后写出正确答案，旁边【📋 复制】按钮可把整题（题型+题号+题干+代码+全部选项+你的答案+正确答案）复制到剪贴板，方便整理错题本。

> 编程题（50 分）需用对应的测试程序自行评测，不计入网页判卷分数。

## 高频考点分布（据真题统计）

1. 变量命名规则 / 关键字识别（几乎每套必考）
2. 基本数据类型与常量类型（int / double / char / bool）
3. 输入输出：`printf` 格式化占位符、`cout`、`scanf` 取地址符、字符 vs 字符串
4. 算术运算：`/` 整除与 `%` 取模、运算优先级
5. 关系 / 逻辑运算：真假判定、`!`、`&&`、`||`、德摩根
6. 三大结构：顺序 / 选择 / 循环；`break` / `continue`；死循环判断
7. 循环执行次数、累加器、计数器
8. 横线填空（补全循环条件 / 整除取模表达式）
9. 代码阅读（手算输出）
10. 计算机常识（冯·诺依曼、图灵、ENIAC、存储设备、操作系统、大模型、摩尔定律等）

各套难度逐步递进，第 10 套为综合冲刺卷。

## 编程题测试程序使用方法

测试程序用于**自动校验考生编写的程序是否正确**。

1. 考生先把自己写的代码编译成可执行文件，例如：
   ```bash
   g++ my_answer.cpp -o my_answer
   ```
2. 进入对应套目录，编译测试程序：
   ```bash
   cd 第01套
   g++ 编程题1_班级平均分_测试程序.cpp -o judge1
   ```
3. 运行测试程序，参数为**考生生成的可执行文件路径**：
   ```bash
   ./judge1 ./my_answer        # Linux / macOS
   judge1.exe my_answer.exe    # Windows
   ```
4. 测试程序会自动喂入多组输入，比对实际输出与期望输出。
   - 全部通过会提示 `ALL PASSED`；
   - 若有不通过，会用**表格**列出该用例的「输入 / 实际输出 / 期望输出」，方便核对正确性。

> 所有评测程序均已用标准解法在 g++ 13.4 下验证通过（每题 10 个测试点）。
