三、单选题解析
第 1 题 A
常量定义后不可修改（A 对）。#define 定义的是宏，不是变量（B 错）；const 变量不可再赋值（C 错）；常量既可用 const 也可用 #define（D 错）。
第 2 题 A
顺序赋值：a=b → a=2；b=c → b=3；c=a → c=2（此时 a 已是 2）。输出 232。这并非正确的三变量交换。
第 3 题 B
(double)(5/2)：先整数除法 5/2=2，再转 double 得 2.0；(double)5/2：先把 5 转 double，再 5.0/2 得 2.5。
第 4 题 A
5/2=2（整除）、5%2=1、5*2=10，输出 2 1 10。
第 5 题 A
后置 n++ 先用原值 10 比较：10>10 为假即 0；比较后 n 自增为 11。输出 0 11。
第 6 题 A
x=8：x>10 否，x>5 是 → 输出 B 并结束整条 if-else 链；之后无条件输出 D。结果 BD。
第 7 题 A
只有 i==j 时累加 i*j：1×1 + 2×2 + 3×3 = 1+4+9 = 14（j 只到 3，i=4 时无匹配）。
第 8 题 B
"同时被 3 和 5 整除"即两个条件都成立，用 &&。等价于被 15 整除。
第 9 题 A
n=5：t = 0*10 + 5%10 = 5，n = 5/10 = 0，循环结束。输出 5。
第 10 题 A
数字逆序经典写法：r 依次得 3 → 32 → 321。这是把 123 倒过来。
第 11 题 A
每轮 i 加 1、j 减 1，二者向中间靠拢。i=5,j=5 时 5<5 不成立停止。输出 5 5。
第 12 题 D
a == a*2 是比较运算（判断相等），不会改变 a 的值，因此不能完成"乘以 2"。A/B/C 都正确赋值。
第 13 题 A
i=1：sum=1；i=2：sum=3；i=3：被 3 整除 continue；i=4：被 4 整除 break。结束时 sum=3。
第 14 题 C
触摸屏既能显示（输出）又能接受触摸操作（输入），是典型的输入输出双重设备。键盘只输入，显示器/打印机只输出。
第 15 题 D
没有语法错误只能保证能编译运行，但逻辑错误仍可能导致结果不正确，D 错误。A、B、C 均正确（for 总能改写为 while）。

五、编程题参考程序
编程题 1：阶梯电价
#include <iostream>
using namespace std;
int main() {
    int x;
    cin >> x;
    double fee;
    if (x <= 200)
        fee = x * 0.5;
    else
        fee = 200 * 0.5 + (x - 200) * 0.8;
    printf("%.2lf\n", fee);
    return 0;
}
要点：分两段计算，超过 200 度的部分单独用 0.8 元/度；注意前 200 度始终按 0.5 计费。用 %.2lf 保留两位小数。
编程题 2：第一个完全平方数
#include <iostream>
using namespace std;
int main() {
    long long N;
    cin >> N;
    long long i = 1;
    while (i * i < N) i++;   // 找到第一个 i 使 i*i >= N
    cout << i * i << endl;
    return 0;
}
要点：从 1 开始枚举 i，找到第一个满足 i*i >= N 的 i，输出 i*i。N 最大 100 万，i 最多约 1000，循环很快。用 i*i 注意不要溢出（long long 稳妥）。
