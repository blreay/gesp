一、单选题 · 详细解析
第 1 题
答案：C（=>）
%=（取余赋值）、--（自减）、!=（不等于）都是 C++ 的合法运算符。=> 不是 C++ 运算符（C++ 中只有 >= 表示"大于等于"，注意符号顺序）。常见陷阱：\= 和 => 都不是 C++ 运算符。
知识点：运算符识别
第 2 题
答案：B（2）
11 / 4 是两个 int 相除，结果为 int 类型，值为 2（截断小数部分）。然后赋值给 double 变量，变成 2.0，但 cout 输出时显示为 2。关键：整数除法在运算时就已经截断了，之后赋给 double 也无法恢复小数部分。要得到 2.75 需要写 11.0 / 4 或 (double)11 / 4。
知识点：整除vs浮点除法 + int截断
第 3 题
答案：D（11）
运算符优先级：% 和 * 同级，从左到右计算，优先于 +。
计算过程：15 % 4 = 3，3 * 3 = 9，2 + 9 = 11。
常见错误：误以为先算 4 * 3 或者忽略 % 和 * 的同级关系。
知识点：运算符优先级（含%混合表达式）
第 4 题
答案：A（5）
29 % 6：29 / 6 = 4 余 5，所以 29 % 6 = 5。
验证：4 x 6 = 24，29 - 24 = 5。
知识点：取余运算
第 5 题
答案：C（20）
x *= 2 + 3; 等价于 x = x * (2 + 3);，即 x = 4 * 5 = 20。
关键陷阱：复合赋值运算符右边的表达式会先完整计算，再与左边运算。x *= 2 + 3 不是 x = x*2 + 3 = 11！
知识点：复合运算符
第 6 题
答案：B（x <= 5 && y >= 3）
根据德摩根定律：!(A || B) 等价于 !A && !B。
所以 !(x > 5 || y < 3) = !(x > 5) && !(y < 3) = x <= 5 && y >= 3。
注意选项 C 用的是 ||（或），D 用的也是 ||，只有 B 正确使用了 &&（与）。
知识点：德摩根定律
第 7 题
答案：D（n % 5 == 0 && n % 3 != 0）
"能被5整除"用 n % 5 == 0，"不能被3整除"用 n % 3 != 0，两个条件同时满足用 &&。
选项 A 用了 ||（或），不满足"同时"要求；B 的条件含义反了；C 用了 / 而非 %。
知识点：逻辑条件组合
第 8 题
答案：B（G）
字符 'D' 的 ASCII 码为 68，68 + 3 = 71，71 对应字符 'G'。
通过 (char) 强制转换后输出字符而非数字。ASCII 码中大写字母连续排列：A=65, B=66, C=67, D=68, E=69, F=70, G=71。
知识点：char运算/ASCII码
第 9 题
答案：A（2 2.25）
(double)(9 / 4)：先算 9/4=2（整数除法），再转 double 得 2.0，输出 2。
(double)9 / 4：先把 9 转成 double 9.0，再除以 4，得 2.25。
关键区别：强制类型转换的括号位置决定了转换发生在除法之前还是之后。
知识点：类型转换（强制转换位置）
第 10 题
答案：C（12）
逐步跟踪：
i=1：1%3=1 (不跳过)，i!=7，s=0+1=1
i=2：2%3=2 (不跳过)，i!=7，s=1+2=3
i=3：3%3=0 (continue跳过)
i=4：4%3=1 (不跳过)，i!=7，s=3+4=7
i=5：5%3=2 (不跳过)，i!=7，s=7+5=12
i=6：6%3=0 (continue跳过)
i=7：7%3=1 (不跳过)，i==7 (break退出)
最终 s=12。
知识点：break/continue混合
第 11 题
答案：B（BC）
k=2，直接跳到 case 2，输出 "B"。由于 case 2 后没有 break，发生 fall-through（穿透），继续执行 case 3，输出 "C"。case 3 后有 break，退出 switch。
注意：switch 中如果不写 break，会从匹配的 case 一直往下执行（穿透），直到遇到 break 或 switch 结束。
知识点：switch fall-through
第 12 题
答案：C（Score:08%）
%02d 表示输出整数，至少 2 位，不足前面补 0。8 只有 1 位，补 0 变成 08。
%% 在 printf 中是转义字符，输出一个 %。
所以最终输出 Score:08%。
知识点：printf %02d补零 + %%转义
第 13 题
答案：C（王选院士是汉字激光照排系统的发明人）
A 错：第三代计算机的核心元件是集成电路（IC），不是电子管。电子管是第一代。
B 错："银河一号"是中国第一台亿次巨型计算机（超级计算机），不是个人电脑。
C 对：王选院士发明了汉字激光照排系统，被称为"当代毕昇"。
D 错：第四代计算机使用的是大规模/超大规模集成电路（VLSI），晶体管是第二代。
四代对应：电子管 → 晶体管 → 集成电路 → 大规模集成电路。
知识点：计算机发展代次 + 中国超算 + 中国IT名人
第 14 题
答案：D（.cn 是中国的顶级域名）
.cn 是中国的国家顶级域名（country-code TLD）。
A 错：.com 是商业机构通用顶级域名，不是中国特有的。
B 错：IP 地址与域名通过 DNS（域名系统）对应。域名就是 IP 的"人类可读"形式。
C 错：域名主要由英文字母、数字和连字符组成。
知识点：网络知识（域名、IP地址）

Windows 常用快捷键：

二、判断题 · 详细解析
判断第 1 题
答案：A（正确 ✔）
flag = false，!flag = !false = true，所以 if(!flag) 的条件为真，会执行 if 里的语句。
! 是逻辑取反运算符，!true = false，!false = true。
知识点：! 取反
判断第 2 题
答案：A（正确 ✔）
if(a = 0) 是赋值语句而非比较。它将 0 赋给 a，整个赋值表达式的值为 0。C++ 中 0 等价于 false，所以条件永远为假。
对比：if(a = 5) 赋值后表达式值为 5（非0），条件永远为真。
正确的比较应写 if(a == 0)（双等号）。
知识点：== vs =
判断第 3 题
答案：B（错误 ✘）
cout << (5 == 5); 输出的是 1，不是 True。
C++ 中 bool 值用 cout 输出时默认显示为整数：true 输出 1，false 输出 0。
另外，C++ 是大小写敏感的：true/false 是关键字，但 True/False 不是。
知识点：bool输出 + C++大小写敏感
判断第 4 题
答案：A（正确 ✔）
当 i 增长到 3 时，if(i == 3) continue; 会跳过本次循环剩余代码（包括 i++），直接回到 while 条件判断。此时 i 仍然是 3，再次进入循环体，再次 continue... 永远无法递增，形成死循环。
教训：在 while 循环中使用 continue 时，要确保循环变量的更新不会被跳过。for 循环中 continue 不会跳过更新表达式，所以更安全。
知识点：continue可能导致while死循环
判断第 5 题
答案：B（错误 ✘）
while(n--) 先判断 n 的值，再减 1。n=3 时：判断 3（真），执行循环体(n变2)；判断 2（真），执行(n变1)；判断 1（真），执行(n变0)；判断 0（假），退出。循环体执行 3 次。
while(--n) 先减 1，再判断。n=3 时：n变2，判断 2（真），执行；n变1，判断 1（真），执行；n变0，判断 0（假），退出。循环体执行 2 次。
3 次 vs 2 次，不相同。
知识点：while(n--) vs while(--n)
判断第 6 题
答案：A（正确 ✔）
异或（XOR）交换法是一种不需要临时变量的交换方式：
设 a=m, b=n。
a = a ^ b; → a = m^n
b = a ^ b; → b = (m^n)^n = m（异或自身为0，m^0=m）
a = a ^ b; → a = (m^n)^m = n
最终 a=n, b=m，交换成功。
知识点：变量交换（异或法）
判断第 7 题
答案：A（正确 ✔）
C++ 中逗号表达式 (a, b, c) 的值是最后一个表达式 c 的值。
int x = (5, 10, 15); 中，逗号表达式依次计算 5、10、15，最终取最后一个值 15 赋给 x。
注意：逗号表达式在实际编程中不常用，但考试中是常考知识点。
知识点：逗号表达式
判断第 8 题
答案：A（正确 ✔）
n=11 的因数只有 1 和 11，所以 cnt=2。
代码逻辑：遍历 1 到 n，统计能整除 n 的数的个数。质数恰好只有两个因数（1 和自身），所以 cnt==2 时为质数。
11 是质数，cnt=2，输出 "YES"。
知识点：质数判断代码
判断第 9 题
答案：B（错误 ✘）
n=2357，逐位提取各位数字并求和：
2357%10=7, sum=7, n=235
235%10=5, sum=12, n=23
23%10=3, sum=15, n=2
2%10=2, sum=17, n=0（退出）
最终 sum=17，不是 14。2+3+5+7=17。
这段代码是经典的"数位分离"模式：n%10 取最低位，n/=10 去掉最低位。
知识点：数位操作（while循环提取各位）
判断第 10 题
答案：B（错误 ✘）
scanf("%d", &x) 的 %d 格式符用于读取 int 类型数据。如果 x 是 double 类型，应使用 %lf。
使用 %d 读取 double 变量会导致类型不匹配，读入的数据不正确（可能是乱码/垃圾值）。
对比：cin >> x; 会根据 x 的类型自动匹配，不存在这个问题。
知识点：scanf/cin类型匹配
