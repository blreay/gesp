第1题 A：#1 运算符识别
逐组分析：
- A: >>=（右移赋值运算符，合法）、<<=（左移赋值运算符，合法）、!=（不等于，合法）。全部合法 ✓
- B: => 不是 C++ 运算符（不要和 >= 混淆），不合法 ✗
- C: ** 不是 C++ 运算符（C++ 没有幂运算符，Python 才有），不合法 ✗
- D: := 不是 C++ 运算符（Pascal/Go 的赋值符号），不合法 ✗
进阶点：>>= 和 <<= 是位移赋值运算符，如 x >>= 2 等价于 x = x >> 2，虽然不常用但确实是合法的 C++ 运算符。
第2题 B：#2 整除vs浮点除法
按运算符优先级（*、/、% 同级，从左到右结合）：
① 2 / 3：两个都是 int，int 除法 = 0（2 除以 3 商 0 余 2）
② 0 * 4.0：int(0) 乘 double(4.0)，int 提升为 double，结果 = 0.0
③ 1 + 0.0：= 1.0（在 cout 输出时显示为 1）
陷阱：很多人以为 2/3 = 0.666... 然后 0.666*4.0 = 2.666，忘了 int/int 先截断。关键在于 2/3 这一步已经得到 0，后面再乘 4.0 也挽回不了。
第3题 C：#3 运算符优先级
含 6 种运算符：%、*、/、-、+、>、&&。优先级从高到低：% * / > + - > > > &&。
① 15 % 4 = 3
② 3 * 2 = 6
③ 8 / 3 = 2（int 除法）
④ 20 - 6 + 2 = 16
⑤ 16 > 5 = true = 1
⑥ 1 && true = 1（true）
陷阱：容易忘记 && 也是运算符且优先级最低；或者误以为 8/3=2.67（int 除法截断）。
第4题 D：#4 取余运算
% 运算符从左到右结合：
① 100 % 7 = 2（因为 100 = 7 * 14 + 2）
② 2 % 3 = 2（因为 2 < 3，所以 2 / 3 = 0 余 2）
陷阱：第二步 2 % 3 容易算错。当被除数 < 除数时，余数就是被除数本身。也有人会先算 7 % 3 = 1 再算 100 % 1 = 0，这是错误的结合方向。
第5题 B：#5 复合运算符
赋值运算符从右到左结合：
① 先算 b %= c：b = 17 % 5 = 2
② 再算 a += b：a = 10 + 2 = 12
最终 a=12, b=2, c=5。
陷阱：如果误以为从左到右，先算 a += b 得 a=27，再算 b %= c 得 b=2，那 a 就是 27（选项 D）。赋值类运算符都是右结合的！
第6题 C：#6 德摩根定律
设 a=3, b=5, c=0，逐步解析：
① a > b = 3 > 5 = false
② !(a > b) = !false = true
③ !(a > b) || c = true || 0 = true（0 视为 false，true || false = true）
④ !!(true) = !false = true = 1
陷阱：双重取反 !! 的效果是将任何非零值转为 1，零转为 0。容易在第 ③ 步误以为 c=0 会让整个 || 为 false，但 || 只要有一边为 true 就够了。
第7题 D：#7 逻辑条件组合
闰年规则拆解为两个条件的"或"：
条件一：能被 4 整除且不能被 100 整除 → year%4==0 && year%100!=0
条件二：能被 400 整除 → year%400==0
两个条件满足其一即可 → 用 || 连接。
A 错：&& 的优先级高于 ||，所以 A 实际被解析为 year%4==0 || (year%100!=0 && year%400==0)，逻辑不对（如 2023 能被 4 不整除但 A 认为它是闰年...不对，2023%4!=0 所以第一个||左边为false，右边也为false，所以A对2023判断正确。但对于 100，100%4==0 为 true，直接返回 true，但 100 不是闰年）。
B 错：要求同时满足三个条件（能被 4 整除且不能被 100 整除且能被 400 整除），条件矛盾（不能被 100 整除就不可能被 400 整除）。
C 错：要求被 400 整除才算闰年，2024 这样的普通闰年会被排除。
D 正确：(year%4==0 && year%100!=0) || year%400==0
陷阱：A 选项利用 && 优先级高于 || 来迷惑，如果不加括号，运算顺序可能不符合题意。
第8题 A：#8 !取反与隐式bool转换
逐个分析：
① !0：0 是 false，!false = true = 1
② !!5：5 是非零（true），!true = false，!false = true = 1
③ !(-1)：-1 是非零（true），!true = false = 0
总和：1 + 1 + 0 = 2
陷阱：关键在第 ③ 步：-1 虽然是负数，但在 C++ 中任何非零值都是 true。!(-1) = !true = false = 0。很多人误以为负数取反后变正，或者 !(-1) = 1。
第9题 C：#9 == vs =
关键：if (i = 2) 中的 = 是赋值而非比较！
逐步跟踪：
- i=0: 0<5 true -> 循环体: i = 2（赋值！i 变为 2），2 非零为 true，cnt=1 -> i++: i=3
- i=3: 3<5 true -> 循环体: i = 2（又被赋为 2），cnt=2 -> i++: i=3
- i=3: 3<5 true -> 循环体: i = 2，cnt=3 -> i++: i=3
- ...死循环！i 永远在 2 和 3 之间交替，永远小于 5。

每次进入循环体，i 被赋为 2，然后 i++ 变为 3。下次判断 3<5 为 true，又进入循环体，i 又被赋为 2... 无限循环。
陷阱：如果写的是 if (i == 2)（比较），只有 i 恰好等于 2 时才 cnt++，不会修改 i 的值，循环正常结束。但 i = 2 是赋值，每次都把 i 改为 2，导致死循环。这是 C++ 初学者最常犯的错误之一。
第10题 B：#10 int截断链
C++ 中 int() 强制类型转换对浮点数是向零截断（truncate toward zero），不是四舍五入，也不是向下取整：
- int(3.6) = 3（丢弃小数部分 .6）
- int(-3.6) = -3（向零截断，不是 -4！）
- 3 + (-3) = 0
陷阱：如果误以为 int(-3.6) 是向下取整（floor），会得到 -4，答案变成 3+(-4)=-1（选项 A）。C++ 的 int 截断是向零方向的，正数截断等于 floor，但负数截断等于 ceil。
第11题 C：#11 bool运算
每个比较表达式的值为 true(1) 或 false(0)：
- (3 > 2) = true = 1
- (5 < 4) = false = 0
- (1 == 1) = true = 1
- (0 != 0) = false = 0
总和：1 + 0 + 1 + 0 = 2
陷阱：容易误认为 (0 != 0) 为 true（"0 不等于 0"听起来像 false，实际确实是 false），或者忘记 bool 值 true 在算术运算中等于 1。
第12题 A：#12 char运算/ASCII码
这道题的陷阱在于运算符优先级：* 优先于 -！
表达式被解析为 '9' - ('0' * 2)，而不是 ('9' - '0') * 2。
- '0' 的 ASCII 码 = 48
- '9' 的 ASCII 码 = 57
- '0' * 2 = 48 * 2 = 96
- '9' - 96 = 57 - 96 = -39
陷阱：如果误以为先算 '9'-'0'=9，再算 9*2=18（选项 C），就上当了。乘法优先级高于减法，必须先算 '0'*2。这是一道综合考查 ASCII 码知识和运算符优先级的题。
第13题 D：#13 类型提升#14 大小写敏感
C++ 是严格大小写敏感的语言：
- endl（全小写）是标准库定义的换行操纵符 ✓
- Endl、ENDL、endL 都不是标准标识符，编译器会报"未声明的标识符"错误 ✗
只有 D 选项使用了正确的全小写 endl。
进阶知识：endl 不仅输出换行符，还会刷新输出缓冲区（flush），这与直接输出 '\n' 有性能区别。但从编译角度，只要大小写不对就直接报错。
第14题 B：#15 嵌套循环break#24 逗号表达式
关键：break 只退出内层循环。
外层循环 i=1,2,3 各执行一次。每次内层循环：
- j=1: 1%2=1 != 0，不 break，s += i*1 = i
- j=2: 2%2=0 == 0，break，退出内层循环
所以每次内层循环只执行 j=1 这一步：s += i*1。
- i=1: s = 0 + 1 = 1
- i=2: s = 1 + 2 = 3
- i=3: s = 3 + 3 = 6
陷阱：选项 A(12) 是以为内层不会 break 而全部执行的结果；选项 C(30) 是以为 j 也乘了多个值的结果。break 只跳出最近的一层循环。
第15题 A：#19 %d %c 混用#20 %-5d 左对齐
printf("%d %c [%-5d]", 65, 65, 42) 逐个格式化：
① %d 将 65 输出为十进制整数：65
② %c 将 65 输出为字符（ASCII 码 65 = 'A'）：A
③ %-5d 将 42 输出为左对齐、宽度 5 的整数：42_ _ _（42 后补 3 个空格）
合起来：65 A [42_ _ _]
陷阱：- %d 和 %c 对同一个数值 65 的输出完全不同！%d 输出数字，%c 输出字符。
- %-5d 中的 - 表示左对齐（数字在左，右边补空格），默认（无 -）是右对齐（左边补空格）。选项 D 错在 [_ _ _42] 是右对齐的写法。

第1题 √ (正确)：#25 计算机代际
计算机四代划分（按核心电子元件）：
- 第一代（1946-1958）：电子管（真空管）
- 第二代（1959-1964）：晶体管
- 第三代（1965-1970）：集成电路（IC）
- 第四代（1971至今）：大规模/超大规模集成电路（LSI/VLSI）
题目说"第二代是晶体管，第一代是电子管，第三代是集成电路"，全部正确。
易错点：把"晶体管"和"电子管"搞混，或者把第三代误记为"大规模集成电路"（那是第四代）。
第2题 √ (正确)：#26 中国计算机
天河二号（Tianhe-2）由中国国防科技大学研制，2013 年 6 月至 2015 年 11 月连续 6 次获得全球超级计算机 TOP500 排行榜第一名。
易错点：容易和"天河一号"（也曾获第一但次数较少）、"神威·太湖之光"（后来接替了天河二号的第一名位置）混淆。
第3题 × (错误)：#27 IT名人细节
"图灵奖"确实是计算机科学领域的最高奖项（被誉为"计算机界的诺贝尔奖"），但它是由 ACM（美国计算机协会）颁发的，不是诺贝尔基金会。诺贝尔基金会颁发的是诺贝尔奖（物理、化学、医学、文学、和平、经济学），没有"诺贝尔计算机奖"。
易错点：因为图灵奖经常被称为"计算机界的诺贝尔奖"，就误以为它是诺贝尔基金会颁发的。这只是一个比喻，不是事实。
第4题 √ (正确)：#28 网络分层
DNS（Domain Name System，域名系统）的核心功能就是将人类可读的域名（如 www.baidu.com）解析（翻译）为计算机能理解的 IP 地址（如 220.181.38.148）。
没有 DNS，用户就只能通过记忆 IP 地址来访问网站，这对人类来说非常不方便。
易错点：误以为 DNS 是"防火墙"或"路由器"的功能，其实 DNS 是一个独立的域名解析服务。
第5题 × (错误)：#29 快捷键
题目故意把两个快捷键的功能对调了：
- Ctrl+S 的功能是保存（Save），不是撤销
- Ctrl+Z 的功能是撤销（Undo），不是保存
其他常用快捷键：Ctrl+A = 全选（Select All），Ctrl+C = 复制，Ctrl+V = 粘贴，Ctrl+X = 剪切。
易错点：快速阅读时容易忽略题目把两个快捷键的功能对调了。
第6题 √ (正确)：#30 编译过程
C++ 从源代码到可执行程序的完整流程：
① 预处理（preprocessing）：处理 #include、#define 等预处理指令
② 编译（compilation）：将预处理后的源代码翻译成目标文件（.o / .obj）
③ 链接（linking）：将多个目标文件和库文件合并生成可执行文件（.exe / a.out）
题目描述的"编译"和"链接"两个步骤是简化但正确的说法。
易错点：忘记"链接"步骤，以为编译直接生成可执行文件；或者把"预处理"误认为是"编译"的一部分（严格来说预处理是独立步骤）。
第7题 √ (正确)：#16 continue导致死循环#17 for循环省略
逐步跟踪：
- i=1: 1%5=1 != 0，不 continue，i++ → i=2
- i=2: 2%5=2 != 0，i++ → i=3
- i=3: 3%5=3 != 0，i++ → i=4
- i=4: 4%5=4 != 0，i++ → i=5
- i=5: 5%5=0 == 0，执行 continue，跳回 while 条件判断。i++ 被跳过了！i 仍为 5。
- i=5: 再次 5%5==0，continue ... 永远卡在 i=5，死循环！
关键：while 循环中 continue 跳到条件判断处，循环变量的更新（i++）在循环体末尾，被 continue 跳过了。这和 for 循环不同——for 循环的 i++ 在更新表达式中，continue 不会跳过它。
第8题 × (错误)：#18 switch穿透#21 三变量赋值
x=3，进入 case 3，输出 "C"。
case 3 没有 break，穿透到 case 4，输出 "D"。
case 4 后面有 break，跳出 switch。不会继续到 case 5。
实际输出：CD，不是 CDE。
易错点：以为穿透会一直到底。实际上穿透遇到 break 就停止。case 4 后面有 break，所以不会穿透到 case 5。
第9题 √ (正确)：#23 数位反转+判断回文#22 因数统计
输入 N=12321，temp=12321，rev=0：
① temp=12321: rev = 0*10 + 12321%10 = 1, temp = 12321/10 = 1232
② temp=1232: rev = 1*10 + 1232%10 = 12, temp = 1232/10 = 123
③ temp=123: rev = 12*10 + 123%10 = 123, temp = 123/10 = 12
④ temp=12: rev = 123*10 + 12%10 = 1232, temp = 12/10 = 1
⑤ temp=1: rev = 1232*10 + 1%10 = 12321, temp = 1/10 = 0
循环结束。rev=12321 == N=12321，输出 "YES"。
逻辑：temp%10 取最低位，rev*10 + digit 把数字依次"倒序拼接"，最终得到反转后的数。如果反转数等于原数，就是回文数。
第10题 × (错误)：#22 因数个数统计
20 的因数：1, 2, 4, 5, 10, 20，共 6 个，不是 4 个。
代码逻辑正确（遍历 1 到 N，对每个 i 判断 N%i==0），所以输出的是 6 而不是 4。
验证：20%1=0 ✓, 20%2=0 ✓, 20%3=2 ✗, 20%4=0 ✓, 20%5=0 ✓, ..., 20%10=0 ✓, ..., 20%20=0 ✓。cnt = 6。
易错点：漏数因数（比如忘记 1 和 20 本身也是因数），或者把"因数个数"和"质因数个数"搞混（20 的质因数只有 2 和 5 两个）。
