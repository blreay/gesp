第1题 C：#25#26
ENIAC（1946年）是世界上第一台通用电子计算机，使用的核心元件是电子管（真空管），不是晶体管也不是集成电路。中国第一台大型通用电子计算机（104机，1959年）同样使用电子管。
易错点：容易把"电子管"和"晶体管"搞混。电子管是第一代计算机的标志，晶体管是第二代，集成电路是第三代。
第2题 B：#1
逐个分析：
- => 不是C++运算符（C++中没有这个符号）
- \= 不是C++运算符（反斜杠不构成赋值运算符）
- >= 是"大于等于"比较运算符 ✓
- /= 是"除法赋值"复合运算符 ✓
- != 是"不等于"比较运算符 ✓
- +=、%= 都是合法运算符 ✓
A 含 =>（非法），C 含 \=（非法），D 同时含 => 和 \=（两个都非法）。只有 B 全部合法。
易错点：把 => 当成 >= 的另一种写法，或把 \= 当成合法运算符。注意顺序很重要：>= 是合法的，但反过来写 => 就不是。
第3题 D：#2#3
a / 4：a 是 int（值为7），4 是 int，int/int = int，7/4 = 1（向零截断）。
a / b：a 是 int（值为7），b 是 double（值为2.0），int/double 会把 int 提升为 double，7.0/2.0 = 3.5。
由于 *、/、% 优先级高于 +，先算两个除法再相加：1 + 3.5 = 4.5。
易错点：①以为 7/4 = 1.75（忘了 int 除法截断）；②以为 7/2.0 也截断为 3（忘了 int/double 结果是 double）。
第4题 C：#4
N / 5 = 23 / 5 = 4（int 整除），N % 5 = 23 % 5 = 3（取余），4 + 3 = 7。
易错点：取余运算概念不清。N % M 就是 N 除以 M 的余数。23 = 5 x 4 + 3，所以余数是 3。
第5题 B：#5#9
逐步跟踪：x=10 → x+=5 得 x=15 → x-=3 得 x=12 → x%=4 得 x=12%4=0。
(x == 0) 是比较运算（不改变 x 的值），结果为 true。
cout << (x == 0) 输出的是 bool 值，在 C++ 中 bool 输出为整数 1（true）或 0（false），不是字符串 "true"/"false"。
易错点：①混淆 ==（比较）和 =（赋值），如果是 x = 0 则会改变 x 的值；②以为 bool 输出是 "true" 字样。
第6题 A：#6
德摩根定律：!(A && B) 等价于 !A || !B。
这里 A 是 x > 5，B 是 y < 10。
!A = !(x > 5) = x <= 5（取反后大于变小于等于）
!B = !(y < 10) = y >= 10（取反后小于变大于等于）
所以结果为 x <= 5 || y >= 10。
易错点：①忘了 && 取反变 ||（或者反过来）；②取反 > 时写成 < 而不是 <=（边界值）。D 选项用了 &&，违反了德摩根定律。
第7题 D：#7#8
"既是偶数又能被5整除"需要两个条件同时满足，用 &&。
偶数条件：N % 2 == 0，等价于 !(N % 2)（当 N%2 为 0 时，!0 = true）。
能被5整除：N % 5 == 0，等价于 !(N % 5)。
A 用了 ||（或），表示满足任一条件即可，逻辑错误。B 用了 ||，同理。C 中 N % 2 && N % 5 表示两个余数都非零，即"既不是偶数且不能被5整除"，完全反了。
易错点：①把 "且" 写成 ||；②不理解 !(N%2) 等价于 N%2==0。
第8题 C：#12
字符 '5' 的 ASCII 码 = 48 + 5 = 53，字符 '3' 的 ASCII 码 = 48 + 3 = 51。
'5' + '3' 是两个 char 类型相加，会提升为 int 进行运算：53 + 51 = 104。
结果不是数学上的 5+3=8，也不是字符 '8'。
易错点：以为字符加法等于数字加法。在C++中，字符参与运算时使用其ASCII码值。
第9题 A：#10#11
int score = 89.7;：89.7 是 double，赋给 int 类型变量会截断小数部分，score = 89（不是四舍五入！）。
bool pass = (score >= 60);：89 >= 60 为 true，pass = true。
cout << score 输出 89（int 不会显示小数点）。
cout << pass 输出 1（bool 值 true 输出为 1，false 输出为 0，不会输出 "true" 字符串）。
易错点：①以为 int 赋值 89.7 会四舍五入为 90；②以为 bool 输出 "true" 字符串。
第10题 B：#15
逐步跟踪（注意 sum+=i 在判断之前执行）：
i=1: sum=0+1=1, 1%2==1 → continue（跳过 break 判断）
i=2: sum=1+2=3, 2%2==0 → 不 continue, 2==6? 否
i=3: sum=3+3=6, 3%2==1 → continue
i=4: sum=6+4=10, 4%2==0 → 不 continue, 4==6? 否
i=5: sum=10+5=15, 5%2==1 → continue
i=6: sum=15+6=21, 6%2==0 → 不 continue, 6==6? 是 → break
循环结束，sum = 21。
易错点：①以为 continue 会跳过 sum+=i（实际上 sum+=i 在 continue 之前）；②没注意 break 只在偶数时才能执行到（奇数时已经 continue 了）。
第11题 C：#16
当 i % 3 != 0 时执行 continue，跳过 cout。
只有当 i % 3 == 0（即 i 能被 3 整除）时，才会执行 cout 输出 i。
1~8 中能被 3 整除的是：3, 6。所以输出 3 6 。
易错点：①把 continue 的效果搞反，以为是跳过后面所有循环（那是 break）；②D 选项加了 9，但循环条件是 i<=8，9 不在范围内。
第12题 D：#18
x=2，进入 case 2，输出 "B"。
case 2 后面没有 break，所以会"穿透"（fall-through）到 case 3，输出 "C"。
case 3 后面有 break，跳出 switch。不会继续执行 case 4。
最终输出 BC。
易错点：①以为 switch 每个 case 自动停止（忘了 fall-through 机制）；②以为会穿透到 case 4（但 case 3 有 break）。
第13题 A：#19#20
%3d：宽度为 3，数值 5 只有 1 位，左边补 2 个空格，显示为 _ _5。
%2d：宽度为 2，数值 123 有 3 位，超过指定宽度时按实际宽度输出，显示为 123（不会截断）。
中间有逗号分隔，最终输出 _ _5,123。
易错点：①以为 %2d 会截断为 12（宽度是最小宽度，不截断）；②以为是左对齐（默认右对齐）；③printf 格式串必须用双引号，不能用单引号。
第14题 C：#21#24
先看变量交换过程：a=3, b=7。
a = a+b = 10; b = a-b = 10-7 = 3; a = a-b = 10-3 = 7。
交换后：a=7, b=3。
关键：cout << (a, b) 中的 (a, b) 是逗号表达式，它先计算 a（丢弃结果），再计算 b，整个表达式的值为 b 的值，即 3。
易错点：①以为 (a, b) 会输出两个值（实际上逗号表达式只取最后一个值）；②交换逻辑算错。
第15题 B：#22#23
回文数判断思路：把数字的每一位反转，然后跟原数比较。
temp % 10 取出当前最低位，rev = rev * 10 + temp % 10 把低位数字依次放到高位。
例如 N=121：temp=121, rev=0*10+1=1; temp=12, rev=1*10+2=12; temp=1, rev=12*10+1=121。rev==N，是回文数。
A 选项 temp / 10 取的是去掉个位后的数，不是个位本身。C/D 选项用了 N 而不是 temp，但 N 保持不变（用来最终比较），循环中应该操作 temp。
易错点：混淆 % 10（取个位）和 / 10（去掉个位）的作用。这也是质数判断中用 N % i == 0 而非 N / i == 0 的类似易混淆点。

第1题 × (错误)：#27
王选先生的主要贡献是发明了汉字激光照排系统，而不是中文操作系统。该系统彻底改变了中文印刷行业，被誉为"汉字印刷术的第二次发明"。
易错点：将"激光照排系统"误记为"操作系统"或"输入法"等其他发明。
第2题 × (错误)：#28
在域名 www.gesp.ccf.org.cn 中，顶级域名是最后的 .cn（国家顶级域名），不是 gesp。域名从右往左解析：cn 是顶级域名，org 是二级域名，ccf 是三级域名，gesp 是四级域名，www 是主机名。
易错点：把域名中最前面的部分误认为是顶级域名。记住：顶级域名在最右边（最后一个点之后的部分）。
第3题 √ (正确)：#29
Ctrl+C 是复制，Ctrl+V 是粘贴。在新文件夹中粘贴会创建原文件的一个副本（不是移动，原文件仍在）。如果想移动文件，应该用 Ctrl+X（剪切）+ Ctrl+V（粘贴）。
易错点：混淆复制（Ctrl+C）和剪切（Ctrl+X）的区别。
第4题 √ (正确)：#13
float(7) 将整数 7 转为 float 类型，值为 7.0。int(7.0) 再将 7.0 转回 int，截断小数部分（7.0 没有小数部分），结果为 7。
这种转换链在处理用户输入时很实用：先用 float 接收可能的浮点输入，再用 int() 截断得到整数。
易错点：以为类型来回转换会丢失精度。对于整数值来说，int→float→int 不会丢失（只要数值在 float 精度范围内）。
第5题 × (错误)：#14
C++ 是大小写敏感的语言。true（全小写）是 C++ 关键字，表示布尔真值。True（首字母大写）不是 C++ 关键字，编译器不认识，会报错。
同理，int 是关键字但 Int 不是，false 是关键字但 False 不是。
易错点：受 Python 等语言影响，Python 中 True/False 首字母大写，但 C++ 中必须全小写。
第6题 √ (正确)：#17
while(N) 等价于 while(N != 0)。当 N=5 时，条件为 true，进入循环。
循环体中只有 cout << N，没有修改 N 的值，所以 N 永远是 5，条件永远为 true，死循环。
如果循环体中有 N-- 或 N /= 10 等语句修改 N，最终 N 会变为 0，循环才会结束。
易错点：①以为 while(N) 中的 N 会自动减少；②忘记 while(0) 不进入循环这个边界条件。
第7题 × (错误)：#30
int X; 声明了整型变量，cin >> X 期望读入一个整数。当用户输入 "hello"（非数字字符串）时，cin 无法将其解析为整数，会进入失败状态（failbit），X 的值不确定（C++11 起为 0），不可能原样输出 "hello"。
易错点：以为 cin 会自动做类型转换，把非数字字符串存入 int 变量。实际上类型不匹配时 cin 会失败。
第8题 × (错误)：#2#3
3 + 10 / 4：先看优先级，/ 优先于 +，所以先算 10 / 4。
10 和 4 都是 int 类型，int / int = int（向零截断），10 / 4 = 2（不是 2.5！）。
然后 3 + 2 = 5，结果是整数 5，不是 3.25 也不是 5.5。
易错点：以为 10/4 = 2.5 然后 3+2.5 = 5.5 或 10/4 = 0.25 然后 3+0.25 = 3.25。记住：两个 int 相除，结果仍是 int，小数部分直接丢弃。
第9题 √ (正确)：#6#8
这是德摩根定律的另一种形式：!(A || B) 等价于 !A && !B。
德摩根定律有两条：
- !(A && B) = !A || !B（取反后 && 变 ||）
- !(A || B) = !A && !B（取反后 || 变 &&）
对于任意布尔值 a 和 b，这两个表达式的结果总是相同的。
易错点：记反了德摩根定律，以为 !(a||b) = !a || !b（运算符没变）。
第10题 × (错误)：#1
=> 不是 C++ 的合法运算符。虽然 >=（大于等于）是合法运算符，但把两个字符反过来写成 => 就不是了。
\= 也不是 C++ 的合法运算符。虽然 /=（除法赋值）是合法运算符，但 \（反斜杠）在 C++ 中是转义字符前缀，不构成赋值运算符。
两个都不是合法运算符，所以说"都是合法运算符"是错误的。
易错点：把 => 和 >= 搞混，把 \= 和 /= 搞混。
