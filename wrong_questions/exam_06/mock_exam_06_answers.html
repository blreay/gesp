第1题 C：#1 运算符识别
逐行分析这段代码：
- 第1行 a % b：% 是合法的取余运算符，int 对 int 取余完全合法 ✓
- 第2行 a /= b：/= 是合法的复合除法赋值运算符。int d = a /= b; 先将 a 除以 b 并赋值给 a（a变为3），然后将 a 的新值赋给 d ✓
- 第3行 a >= b：>= 是合法的"大于等于"比较运算符 ✓
- 第4行 a => b：=> 不是合法的 C++ 运算符！
陷阱分析：A 选项想让你以为全部合法；B 选项设了陷阱说 /= 不能用于初始化（实际上可以）；D 选项说 % 不能用于 int（完全错误，% 就是用于整数的）。关键是区分 =>（非法）和 >=（合法）——顺序反了就不对。
第2题 B：#2 整除vs浮点除法#13 混合类型提升
int a=7, b=2;
double x = a / b;：a 和 b 都是 int，int / int = int，7/2=3（截断），然后 3 赋给 double 变量 x，x=3（注意 cout 默认输出 double 时不显示 .0 如果值恰好为整数，实际输出为 3）。
double y = a / 2.0;：a 是 int，2.0 是 double，int / double → double，7/2.0=3.5。
输出：3 3.5
关键陷阱："x 是 double 类型"不意味着除法会按 double 来算！类型转换发生在赋值时，不是运算时。a/b 在运算时两边都是 int，结果就是 int，之后才赋给 double。
第3题 A：#3 运算符优先级
int result = a > b ? c : d;
解析优先级：赋值运算符 = 的优先级最低。三元运算符 ?: 优先级高于 =，比较运算符 > 优先级更高。
等价于：result = ((a > b) ? c : d)
- a > b 即 5 > 3 为 true
- 三元运算符取 c 的值，即 8
- result = 8
陷阱：C 选项诱导你以为 a > b 的结果（true=1）就是最终值。实际上三元运算符的意思是"如果条件为真取第一个值，否则取第二个值"。D 选项诱导你以为结果是 a 本身的值。
第4题 D：#4 取余的数学应用
n = 4567：
- ones = n % 10 = 4567 % 10 = 7（个位）
- tens = n / 10 % 10 = 456 % 10 = 6（十位：先除以10去掉个位，再取余得到十位）
- ones + tens = 7 + 6 = 13
- 13 % 2 = 1 ≠ 0，进入 else 分支，输出 odd
双重陷阱：
① A 选项故意写了"正确推理过程"但最后给了错误结论（说应该输出 odd，但又标注"等等，应该输出 odd"来制造混淆）——这是"看起来像陷阱但其实推理正确，只是表述混乱"的选项。
② C 选项说 tens=5（把 n/10%10 算错了：4567/10=456，456%10=6 不是5）。
③ B 选项用 n 本身的奇偶性来判断，但代码判断的是个位+十位之和的奇偶性，不是 n 本身。
第5题 C：#5 复合运算符
a *= b + c; 等价于 a = a * (b + c);（不是 (a * b) + c）
因为复合赋值运算符的右侧是一个完整的表达式 b + c，先计算完再与 a 相乘。
a = 3 * (4 + 5) = 3 * 9 = 27
经典陷阱：A 选项诱导你以为先算 a*b=12 再加 c=5 得17。这是最常见的错误理解。记住：a *= expr 等价于 a = a * (expr)，右边的表达式会先完整计算。
第6题 B：#6 德摩根定律
if 条件是 score >= 60 && age < 18。进入 else 意味着 if 条件为假。
根据德摩根定律：!(A && B) = !A || !B
- A = score >= 60，!A = score < 60（注意 >= 取反是 <）
- B = age < 18，!B = age >= 18（注意 < 取反是 >=）
所以 else 条件是：score < 60 || age >= 18
陷阱分析：
① A 选项用了 &&（应该是 ||，德摩根定律取反后 && 变 ||）
② C 选项写了 age > 18（边界错误：< 18 取反应该是 >= 18，不是 > 18）
③ D 选项同时犯了用 && 和边界值两个错误
第7题 A：#7 && 和 || 优先级
C++ 中 && 的优先级高于 ||。
r1 = a || b && c 等价于 a || (b && c) = true || (false && false) = true || false = true (1)
r2 = (a || b) && c = (true || false) && false = true && false = false (0)
输出：1 0
陷阱：如果不知道 && 优先级高于 ||，会以为 a || b && c 等于 (a || b) && c，得到错误答案 0 0。
第8题 B：#8 ! 运算#9 赋值在条件中
while (n = n / 10) 这里是赋值（=），不是比较（==）。
每次循环先计算 n/10，赋值给 n，然后将 n 的新值作为 while 的条件（非零为真，零为假）。
跟踪过程：
- 第1次：n = 4567/10 = 456（非零，进入循环），count=1
- 第2次：n = 456/10 = 45（非零），count=2
- 第3次：n = 45/10 = 4（非零），count=3
- 第4次：n = 4/10 = 0（为零，循环结束）
输出：3
陷阱：A 选项诱导你以为"4位数字→循环4次"，但第4次时 n 变为 0，条件不满足，循环体不执行。D 选项诱导你以为条件中不能赋值（实际上 C++ 允许，只是编译器可能发出警告）。
第9题 C：#10 多重截断
int(3.9) = 3（截断小数部分，不是四舍五入！）
int(2.1) = 2（同样截断）
a = 3 * 2 = 6

3.9 * 2.1 = 8.19（先做浮点乘法）
int(8.19) = 8（再截断）
b = 8

输出：6 8
核心陷阱：int(3.9) * int(2.1) 和 int(3.9 * 2.1) 结果不同！前者先截断再乘（3*2=6），后者先乘再截断（8.19→8）。这是"截断的时机"问题。
第10题 D：#21 异或交换#26 二进制运算
异或交换三步法（不使用临时变量交换两个值）：
初始：a=13 (二进制 1101), b=7 (二进制 0111)

第1步 a ^= b：a = 1101 ^ 0111 = 1010 = 10
第2步 b ^= a：b = 0111 ^ 1010 = 1101 = 13
第3步 a ^= b：a = 1010 ^ 1101 = 0111 = 7

结果：a=7, b=13（交换成功！）
原理：异或有"自反性"：x ^ y ^ y = x。三步异或操作本质是利用这个性质完成交换。
陷阱：B 选项只跟踪了第一步就停了；C 选项以为异或会互相抵消为0。
第11题 B：#22 最大公约数
这是辗转相除法（欧几里得算法）求最大公约数。逐步跟踪：
初始：a=48, b=18

第1轮：t=18, b=48%18=12, a=18 → (a=18, b=12)
第2轮：t=12, b=18%12=6, a=12 → (a=12, b=6)
第3轮：t=6, b=12%6=0, a=6 → (a=6, b=0)

b=0，while(b) 条件为假，循环结束。输出 a = 6
验证：48 = 6 x 8, 18 = 6 x 3，GCD(48,18) = 6 ✓
陷阱：C 选项（18）是以为循环只执行一轮就结束了；A 选项（2）是算错了取余。
第12题 A：#23 数位求和#24 复杂循环
外层 for 循环遍历 num = 38, 39, 40。内层 while 循环对每个 num 求各位数字之和。

num=38：3+8 = 11
num=39：3+9 = 12
num=40：4+0 = 4

total = 11 + 12 + 4 = 27
陷阱：B 选项把 40 的数位和算成了 7（把 40 当成 4+0=4 算对了但总和加错了，或者把 40 当成个位0十位4百位0漏算）；D 选项把 39 的数位和少算了一位。
第13题 B：#15 三层嵌套循环中的break
三层循环中，break 只能跳出最内层的循环（k 循环），外面两层（i 和 j 循环）继续执行。

对于每一对 (i, j)：
- k=0：k!=1，执行 count++（count加1）
- k=1：k==1，执行 break，跳出 k 循环

所以每次内层循环只执行 k=0 这一次，count 加 1。
i 有 3 个值 (0,1,2)，j 有 3 个值 (0,1,2)，共 3 x 3 = 9 次。

核心知识点：break 只跳出直接包含它的那一层循环！
陷阱：A 选项以为 break 只执行了一个 i 循环（3次）；D 选项以为第一次 break 就终止了所有循环。
第14题 D：#16 while(n--)执行次数
n-- 是后置递减：先取 n 的当前值作为表达式的值（用于条件判断），然后再将 n 减 1。

跟踪：
- 判断 n=5（非零=true），n 变为 4，count=1
- 判断 n=4（非零=true），n 变为 3，count=2
- 判断 n=3（非零=true），n 变为 2，count=3
- 判断 n=2（非零=true），n 变为 1，count=4
- 判断 n=1（非零=true），n 变为 0，count=5
- 判断 n=0（零=false），n 变为 -1，循环结束

循环体执行 5 次，最终 n = -1。输出：5 -1
陷阱：A 选项以为 n 最终为 0（忘了最后一次判断时 n=0 还会执行 n--）；B 选项混淆了前置递减 --n（那样才是先减再判断，只循环4次）。
第15题 C：#19 %d输出char#20 printf返回值
printf("%d", 'A')：
- 'A' 的 ASCII 码是 65
- %d 按整数输出，所以打印出 65（两个字符）
- printf 的返回值是成功输出的字符个数，"65" 是 2 个字符，所以 r = 2

接着 cout << " " << r 输出空格和2。
最终输出：65 2
双重陷阱：
① A 选项以为 %d 会输出字符 A（%d 是按整数格式输出，要输出字符应该用 %c）
② B 选项知道输出 65，但以为 printf 返回 1（以为是"输出了1个参数"，实际是"输出了2个字符"）

第1题 ✓ (正确)：#25 冯·诺依曼体系
冯·诺依曼体系结构（1945年提出）确实包含五大部件：运算器（ALU）、控制器（CU）、存储器（Memory）、输入设备（Input）和输出设备（Output）。其中运算器和控制器合在一起组成了 CPU（中央处理器）。
这道题看起来像是"太简单肯定有陷阱"，但实际上就是正确的。这就是"陷阱中的陷阱"——让你因为怀疑而选错。
第2题 ✗ (错误)：#27 图灵测试
图灵测试（Turing Test）是由计算机科学之父阿兰·图灵在 1950 年提出的，用于判断机器是否具有智能（能否表现出与人类无法区分的行为），而不是测试计算机运算速度。
测试方法：一个人通过文字与机器和另一个人分别对话，如果无法分辨哪个是机器，则认为该机器通过了图灵测试。
易错点：把"图灵测试"和"性能测试（benchmark）"搞混。
第3题 ✓ (正确)：#28 DNS作用
DNS（域名系统）的核心作用就是将人类可读的域名（如 www.baidu.com）解析为计算机可识别的 IP 地址（如 110.242.68.66）。没有 DNS，我们就必须记住每个网站的 IP 地址才能访问。
这道题也是"正确的"——故意设计让考生在一连串正确/错误交替的题目中容易判断失误。
第4题 ✗ (错误)：#29 操作系统举例
Windows、Linux、macOS 都是操作系统 ✓
但 Microsoft Word 是应用软件（文字处理软件），不是操作系统 ✗
操作系统是管理计算机硬件和软件资源的系统软件，为应用程序提供运行环境。Word 是运行在操作系统之上的应用程序。
易错点：容易快速扫一眼题目，看到前三个都是操作系统就选正确，忽略了最后一个 Word。
第5题 ✓ (正确)：#30 编译错误vs运行错误
int a = 10 / 0; 在语法上是合法的 C++ 代码（编译器可以编译通过，虽然可能发出警告），但在运行时会因为除以零导致未定义行为（通常程序崩溃）。
编译错误（compile error）：语法不符合 C++ 规则，如 int 3x = 5;（变量名以数字开头）。
运行错误（runtime error）：语法正确但执行时出问题，如除以零、数组越界等。
注意：这题是正确的！虽然编译器可能发出"除以零"的警告，但警告不是错误，代码仍然可以编译通过。
第6题 ✗ (错误)：#11 bool的本质
在 C++ 中，true 是 bool 类型的值，sizeof(bool) = sizeof(true) = 1（1个字节），不是4。
虽然 true 在参与整数运算时会被提升为 int（值为1），但 sizeof 检查的是原始类型的大小，true 的类型是 bool，大小为 1。
注意区别：在 C 语言中，sizeof(true) 确实是 4（因为 C 语言中 true 定义为 int 类型的 1）。但 C++ 中 true 是 bool 类型的字面量。
第7题 ✗ (错误)：#12 char溢出
char 类型在大多数平台上是 signed char（有符号），范围是 -128 ~ 127。
赋值 200 超出了这个范围，会发生溢出。200 的二进制是 11001000，作为 signed char 解释：最高位是符号位（1=负），值为 200 - 256 = -56。
所以 (int)c 输出的是 -56，不是 200。
深层知识：如果用 unsigned char c = 200; 则可以正确存储 200（无符号 char 范围是 0~255）。
第8题 ✗ (错误)：#14 标识符命名
逐个分析5个标识符：
- _total：以下划线开头，合法 ✓
- for：是 C++ 关键字，不能作为变量名 ✗
- x2y：以字母开头，包含字母和数字，合法 ✓
- 2nd_place：以数字开头，不合法 ✗
- my-var：包含连字符 -（减号），不合法 ✗

合法的只有 2 个（_total 和 x2y），不是3个。
陷阱：容易把 my-var 误认为合法（以为连字符像下划线一样可以用于变量名），或者忘了 for 是关键字。
第9题 ✓ (正确)：#17 死循环写法
for(;;) 是合法的 C++ 写法，三个表达式（初始化、条件、更新）全部省略。省略条件时默认为 true，所以是死循环 ✓
while(1) 条件永远为真（1 是非零值），也是死循环 ✓
两种写法在 C++ 中都是标准的死循环写法，完全合法。
"看起来像陷阱"：for 的三个部分全部空着看起来很奇怪，可能让人以为是语法错误，但 C++ 标准明确允许这种写法。
第10题 ✗ (错误)：#18 switch无default
x=5，switch 中只有 case 1、case 2、case 3，没有 default。
当 x=5 不匹配任何 case 时，switch 语句什么都不执行，直接跳过整个 switch 块。
然后执行 switch 后面的 cout << "done";，输出 done。

题目说"会输出 other"，但实际输出是 done（没有 "other"），所以是错误的。
关键：没有 default 分支时，不匹配的值不会触发任何 case 的执行。如果需要在不匹配时输出 "other"，应该加上 default: cout << "other"; break;。
