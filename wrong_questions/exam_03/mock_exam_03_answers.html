一、单选题 答案与解析
第 1 题：以下哪个 C++ 表达式能够通过编译且不报错？
答案：C 运算符识别
解析：
A. 10 % 3.0 —— % 运算符要求两个操作数都是整数类型，3.0 是 double，编译报错。
B. "hello" + 1 —— 字符串字面量加整数得到的是指针（指向第2个字符），不能直接赋给 int。
C. 'A' + 3 —— char 与 int 运算是合法的，'A' 的 ASCII 码为 65，结果为 68，可以赋给 int。正确。
D. 10 % 0 —— 虽然编译可能通过，但运行时除零是未定义行为。本题问"能通过编译且不报错"，C 是最安全无歧义的选项。
第 2 题：int 与 double 混合除法
答案：B 整除 vs 浮点除法
解析：
a / b：a 和 b 都是 int，所以是整数除法。7 / 2 = 3（截断小数）。
a / 2.0：2.0 是 double，所以 a 被提升为 double，浮点除法。7 / 2.0 = 3.5。
result = 3 + 3.5 = 6.5。
陷阱：前半部分 a/b 是整除得 3，不是 3.5！两个 int 相除始终是整除。
第 3 题：运算符优先级长表达式
答案：D 运算符优先级
解析：按优先级依次计算：
1. % 和 * 优先级相同，从左到右：8 % 3 = 2，然后 2 * 2 = 4
2. + 和 -：3 + 4 - 1 = 6
3. >：6 > 4 为 true（即 1）
4. ||：true || false 为 true（即 1）
优先级顺序： % * / > + - > > < > && > ||
第 4 题：负数取余
答案：A 取余运算（负数）
解析：
C++ 中取余运算的结果符号与被除数（左操作数）一致。
(-13) % 5：-13 = (-2)*5 + (-3)，所以结果为 -3。
10 % (-3)：10 = (-3)*(-3) + 1，结果符号跟随被除数 10（正数），所以结果为 1。
-3 + 1 = -2。
关键规则：C++ 中 a % b 的结果符号与 a（被除数）相同。
第 5 题：连续复合赋值运算符
答案：C 复合运算符
解析：赋值运算符是右结合的，从右到左执行：
1. 先算 c *= 2：c = 3 * 2 = 6
2. 再算 b += c（此时 c=6）：b = 5 + 6 = 11
3. 最后算 a -= b（此时 b=11）：a = 10 - 11 = -1
所以 a=-1, b=11, c=6。
陷阱：很容易忘记赋值运算符的右结合性，从左到右算会得到错误结果。
第 6 题：嵌套取反（德摩根定律）
答案：B 德摩根定律 / 嵌套取反
解析：当 x = 0 时：
x > 0 为 false（即 0）。
!!(x > 0) = !!(false) = !(true) = false = 0。
（!! 双重取反，相当于把任何值转为 bool：非零变 1，零仍为 0）
x == 0 为 true。
!(x == 0) = !true = false = 0。
0 + 0 = 0。
关键：!! 不是"变为相反"，而是"标准化为 0 或 1"。!!0 仍然是 0。
第 7 题：三条件的 &&/|| 组合
答案：D 逻辑条件组合
解析：a=3, b=7, c=5
A. 3 > 7 && 7 > 5 = false && true = false
B. 3 > 5 || 7 < 5 = false || false = false
C. 3 > 7 || 7 < 5 && 3 < 5：&& 优先级高于 ||，先算 7 < 5 && 3 < 5 = false && true = false，然后 false || false = false
D. 3 < 7 && (7 > 5 || 3 > 5) = true && (true || false) = true && true = true
注意 C 选项：&& 的优先级高于 ||，不加括号时先算 && 部分。
第 8 题：|| 短路求值
答案：A 短路求值
解析：
x > 3 即 5 > 3 为 true。
因为 || 运算符具有"短路"特性：左边为 true 时，右边不会执行。
所以 ++y 不被执行，y 保持为 10。
if 条件为 true，执行 x += 1，x 变为 6。
结果：x=6, y=10。
对比：如果是 && 运算符，左边为 false 时右边才不执行。|| 则相反——左边为 true 时右边被跳过。
第 9 题：if 中的 = 赋值而非 == 比较
答案：C == vs =
解析：
if (a = b) 是赋值语句，不是比较！
首先 a = b 将 b 的值 3 赋给 a，此时 a 变为 3。
赋值表达式的值就是被赋的值，即 3。
3 作为 if 的条件，非零为 true，所以进入 if 分支。
输出 a 的值，即 3。
经典陷阱：= 是赋值，== 才是比较。if(a = 0) 会赋值且永远为 false，if(a = 非零值) 永远为 true。
第 10 题：int() 截断 + bool 与 int 混用
答案：B 多重截断 / bool与int混用
解析：
int(7.9) = 7（截断，不是四舍五入）
int(0.1) = 0（截断）
(true + true) * false：true 在算术运算中为 1，false 为 0。
(1 + 1) * 0 = 2 * 0 = 0
7 + 0 + 0 = 7。
陷阱：int(0.1) 是 0 而不是 1！int() 是向零截断。同时 * false 让整个乘积为 0。
第 11 题：char 运算 + static_cast
答案：A char运算 / 强制类型转换
解析：
'z' - 'a' + 1：小写字母 z 的 ASCII 码为 122，a 的 ASCII 码为 97。
122 - 97 + 1 = 26（正好是英文字母的个数）。
static_cast<int>(26.9) = 26（向零截断，不是四舍五入）。
26 == 26 为 true，即 1。
关键点：static_cast<int> 和 int() 效果相同，都是截断小数部分。
第 12 题：C++ 大小写敏感
答案：D 大小写敏感
解析：
C++ 是严格区分大小写的语言。
cout 是标准库中定义在 std 命名空间的对象（全小写）。
Cout、COUT、CIN 都不是标准标识符，编译器不认识，会报错。
只有 D 选项 cout << "hello"; 是正确的（假设已有 using namespace std;）。
注意：cout 不是关键字，是标准库对象，但它的拼写必须完全匹配。
第 13 题：嵌套循环中的 break
答案：B 嵌套循环中的break
解析：
外层循环 i 从 0 到 2（3 次）。每次外层循环中，内层循环 j 从 0 开始：
- j=0：j != 2，count++，count 加 1
- j=1：j != 2，count++，count 加 1
- j=2：j == 2，执行 break，只退出内层循环
每次外层循环内层贡献 2，外层 3 次，总共 3 * 2 = 6。
关键：break 只能跳出它所在的最内层循环，外层循环继续正常执行。
第 14 题：printf 混合格式
答案：A printf格式化
解析：
printf("[%d]=%c, %.1f", n, n, n * 1.5);
- [%d]：[ 和 ] 是普通字符，%d 输出 n 的整数值 65 → [65]
- %c：将 n（65）作为字符输出，ASCII 65 是大写字母 A → =A
- %.1f：65 * 1.5 = 97.5，保留一位小数 → 97.5
完整输出：[65]=A, 97.5
注意：[] 等方括号在 printf 中是普通字符，不像 {} 也是普通字符。%c 和 %d 对同一个整数值 65 有不同的输出形式。
第 15 题：三变量轮换 + 逗号运算符在 for 中
答案：C 三变量轮换 逗号运算符
解析：
第一部分：三变量轮换（a→b→c→a 的循环赋值）。
初始值：a=1, b=2, c=3。
temp = a; → temp=1
a = b; → a=2
b = c; → b=3
c = temp; → c=1
轮换后：a=2, b=3, c=1（每个变量都得到了"下一个"的值）。

第二部分：带逗号运算符的 for 循环。
for(int i=0, j=3; i < j; i++, j--) 中，逗号运算符在初始化和更新部分同时操作两个变量。
- i=0, j=3：0<3 ✓，a=2+1=3。更新 i=1, j=2
- i=1, j=2：1<2 ✓，a=3+1=4。更新 i=2, j=1
- i=2, j=1：2<1 ✗，退出
循环执行 2 次，a 从 2 增加到 4。
输出：4 3 1。
陷阱：三变量轮换容易搞混方向。如果忘记 temp，直接赋值 a=b; b=c; c=a; 会导致 a 的原值丢失。
二、判断题 答案与解析
判断第 1 题：计算机四代划分标准
答案：B（错误） 计算机四代划分标准
解析：
计算机的四代划分主要依据是主要电子元件（核心器件），而不是运算速度。
- 第一代：电子管（真空管）
- 第二代：晶体管
- 第三代：集成电路
- 第四代：大规模/超大规模集成电路
运算速度确实在不断提高，但不是划分的标准。
判断第 2 题：二进制 110101 转十进制
答案：B（错误） 二进制与十进制转换
解析：
110101 从右到左：
1×20 + 0×21 + 1×22 + 0×23 + 1×24 + 1×25
= 1 + 0 + 4 + 0 + 16 + 32 = 53，不是 51。
陷阱：51 的二进制是 110011，容易与 110101 混淆。
判断第 3 题：图灵与冯·诺依曼
答案：B（错误） 图灵 / 冯·诺依曼
解析：
题目把两人的贡献说反了。
- 图灵（Alan Turing）被称为"计算机科学之父"，提出了图灵机理论。
- 冯·诺依曼（von Neumann）提出了"存储程序"的计算机设计思想（冯·诺依曼体系结构）。
题目颠倒了两人的身份和贡献，所以错误。
判断第 4 题：URL 结构
答案：A（正确） URL/域名结构
解析：
URL 的标准格式为：协议://域名/路径
在 https://www.example.com/page 中：
- https 是协议（超文本传输安全协议）
- www.example.com 是域名
- /page 是路径
题目描述完全正确。
判断第 5 题：重命名文件
答案：B（错误） 文件操作
解析：
"重命名"操作是修改文件的名称，原文件名不再存在，取而代之的是新名称。文件内容不变，位置不变，但名字变了——这意味着原文件名已经不存在了，不是"仍然保留在原位置不变"。
对比："复制"才会在保留原文件的同时产生一个新文件。"移动"会改变文件位置。
判断第 6 题：编译型 vs 解释型语言
答案：A（正确） 编译型 vs 解释型语言
解析：
C++ 是典型的编译型语言，源代码（.cpp）需要经过编译器编译、链接后生成可执行文件才能运行。
Python 是典型的解释型语言，由解释器逐行读取并执行，不需要预先编译成机器码。
题目描述正确。
判断第 7 题：continue 在 while 中的陷阱
答案：A（正确） continue在while中的陷阱
解析：
当 i 增长到 5 时，if (i == 5) 为真，执行 continue。
continue 会跳过本次循环的剩余部分（包括 i++），直接回到循环条件判断。
此时 i 仍为 5，条件 i < 10 仍为真，再次进入循环体，又遇到 continue……
这就形成了死循环！i 永远停在 5，永远跳过 i++。
关键区别：在 for 循环中，continue 会跳到更新表达式（如 i++），不会死循环。但在 while 中，如果计数器更新在 continue 之后，就会被跳过。
判断第 8 题：for(;;) 与 switch default 位置
答案：B（错误） for循环省略部分 switch default位置
解析：题目包含两个说法，都是错误的。

说法1：for(;;) 不会导致编译错误。for 语句的三个部分（初始化、条件、更新）都可以省略。省略条件时默认为 true，所以 for(;;) 等价于 while(true)，是合法的无限循环写法。

说法2：switch 中的 default 分支不必写在最后面。default 可以出现在任何 case 之间，甚至可以写在最前面。编译器会在所有 case 都不匹配时跳转到 default，无论它的位置在哪里。

两个说法都是错的，所以整个判断题为错误。
判断第 9 题：数位求积
答案：B（错误） 数位求和/求积
解析：
这段代码求的是各位数字之积（product，乘积），不是和。
n = 1234，各位数字为 1、2、3、4。
逐步跟踪：
- n=1234：1234%10=4，product=1*4=4，n=123
- n=123：123%10=3，product=4*3=12，n=12
- n=12：12%10=2，product=12*2=24，n=1
- n=1：1%10=1，product=24*1=24，n=0
输出为 24，不是 15。（15 是各位数字之和 1+2+3+4=10... 也不对。数位之和是 10，数位之积是 24。）
陷阱：容易把"积"看成"和"。题目说输出 15 是错误的。
