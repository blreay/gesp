第1题 B：#1 运算符识别#14 标识符规则
逐项分析 C++ 标识符规则：
- A. int 2nd_place = 2; — 变量名以数字 2 开头，不合法。C++ 标识符必须以字母或下划线开头。
- B. int _MAX_SIZE = 100; — 以下划线开头，后跟大写字母和下划线，合法。下划线开头、数字结尾、全大写都是合法的标识符。
- C. int x+y = 5; — + 是运算符，不能出现在变量名中，不合法。编译器会把 x+y 解析为表达式 x + y。
- D. int my var = 3; — 变量名中间有空格，不合法。编译器会把 my 和 var 当作两个独立的标识符。
易错点：以为下划线开头不合法，或以为全大写不合法。C++ 的合法标识符规则是：以字母或下划线开头，后续可以是字母、数字或下划线。
第2题 C（值为1）：#2 整除vs浮点#3 优先级#13 转换位置
这道题是知识点交叉的核心题，考查强制转换位置不同导致结果不同：
- (int)(3.5 + 4.5)：先加后转。3.5 + 4.5 = 8.0，然后 (int)8.0 = 8。
- int(3.5)：先转。3.5 截断为 3。
- int(4.5)：先转。4.5 截断为 4。
- 最终：8 - 3 - 4 = 1。
关键公式：(int)(a + b) ≠ int(a) + int(b)，因为截断操作不满足分配律。前者先求精确和再截断，后者分别截断再求和。
易错点：以为先转后转结果一样（选A=0）。这是一个经典的"截断分配律陷阱"。
第3题 D（值为7）：#3 三目运算符#8 !取反
逐步计算：
- !0：0 是 false，取反为 true = 1。
- !1：1 是 true（非零即真），取反为 false = 0。
- !!2：先 !2，2 非零即真，取反为 false = 0；再 !0 = true = 1。
- 3 > 2 ? 5 : 10：3 > 2 为 true，选 5。
- 相加：1 + 0 + 1 + 5 = 7。
注意运算符优先级：!（逻辑非）优先级很高，高于算术运算符 +；三目运算符 ?: 优先级很低，低于 +，但这里用了括号所以不影响。如果没有括号写成 !0 + !1 + !!2 + 3 > 2 ? 5 : 10，由于 + 和 > 优先级高于 ?:，结果会不同。
易错点：①以为 !1 = -1（逻辑非的结果只有 0 和 1）；②以为 !!2 = 2（双重否定恢复的是逻辑值 1，不是原值）；③忘记三目运算符的求值规则。
第4题 B（输出5）：#4 取余运算（周期性）
逐步计算：
- today = 3（周三），days = 100。
- days % 7 = 100 % 7：7 × 14 = 98，100 - 98 = 2。即100天包含14个完整周多2天。
- (today + days % 7) % 7 = (3 + 2) % 7 = 5 % 7 = 5。
- future = 5，不等于 0，不需要调整。
- 输出 5（周五）。
取余运算的周期性应用：星期几是一个典型的周期为 7 的问题。在当前星期数上加上经过的天数，再对 7 取余，就得到未来的星期数。注意 result == 0 时需要特殊处理为 7（表示周日）。
易错点：①直接算 103 % 7 不分步，容易算错；②忘记先对 100 取余再加，虽然结果一样但容易混淆。
第5题 B（输出6）：#5 复合运算符
核心规则：a /= b + 1 等价于 a = a / (b + 1)，不是 a = a / b + 1。
复合赋值运算符 /= 的右边是一个完整的表达式 b + 1，会先计算右边的值，再执行除法赋值。
- b + 1 = 3 + 1 = 4
- a = a / 4 = 24 / 4 = 6
易错点：
- 选 A（8）：误以为 a /= b 再 + 1 被忽略，24/3=8。
- 选 C（9）：误以为 a = a/b + 1 = 24/3 + 1 = 8 + 1 = 9，把复合运算符拆开理解错了。
- 选 D（7）：把 /= 理解成 a = a / b + 1 = 24/3 + 1 但 24/3=7 算错了。
第6题 C：#6 德摩根定律#7 &&高于||
首先确定运算顺序：&& 优先级高于 ||，所以原式等价于：
!((a > 0 && b > 0) || c > 0)
设 X = a > 0 && b > 0，Y = c > 0，则原式为 !(X || Y)。
第一步德摩根：!(X || Y) = !X && !Y
第二步德摩根（展开 !X）：!X = !(a > 0 && b > 0) = (a <= 0 || b <= 0)
!Y = !(c > 0) = (c <= 0)
最终：(a <= 0 || b <= 0) && c <= 0，即选项 C。
易错点：
- 选 A：忘记 && 变 ||、|| 变 &&，全部用 || 连接。
- 选 B：不理解优先级，把原式当成 !(a > 0) && !(b > 0) || !(c > 0) 然后全用 && 连接。
- 选 D：把 && 和 || 的分组搞反了。
第7题 C（输出10）：#9 ==vs=#11 bool算术
关键陷阱：if (x = 0) 中 = 是赋值运算符，不是比较运算符 ==！
- x = 0 先将 x 赋值为 0，然后整个赋值表达式的值为 0。
- if(0) 等价于 if(false)，条件为假，执行 else 分支。
- else 分支：true * 10 + false * 5。
- 在 C++ 中，true 参与算术运算时提升为 int 值 1，false 提升为 0。
- 1 × 10 + 0 × 5 = 10 + 0 = 10。
易错点：
- 选 A（2）：把 = 看成 ==，以为 x==0 为 false（x 初值是 5），进入 else... 不对，x==0 为 false 也进入 else。但如果以为 x==0 为 true 进入 if，则输出 true+true=2。
- 选 B（0）：以为输出 x 的值。
- 选 D（5）：以为 false 在算术中也等于 1，或 false*5=5。
第8题 B（输出-1）：#10 int截断向零#12 char相减
这道题交叉考查两个易错知识点：
1. char 相减求字母间距：
- 'G' 的 ASCII 码 = 71，'A' 的 ASCII 码 = 65。
- ch1 - ch2 = 71 - 65 = 6（G 是第 7 个大写字母，A 是第 1 个，间距为 6）。
2. int 截断向零（不是向下取整！）：
- int(-7.8) = -7，不是 -8！C++ 的 int 转换是截断小数部分（向零方向取整），即直接丢弃小数，保留整数部分。
- 对于正数：int(3.9) = 3（向下）。
- 对于负数：int(-7.8) = -7（向上，即向零方向）。这是最容易错的地方！
3. 最终计算：6 + (-7) = -1。
易错点：
- 选 A（-2）：以为 int(-7.8) = -8（向下取整），6 + (-8) = -2。这是最常见的错误！
- 选 D（13）：不知道 char 用 ASCII 码参与运算，以为 'G' = 7，'A' = 1。
第9题 B（输出18）：#15 双重循环break#18 switch在for内
核心考点：switch 中的 break 只退出 switch，不退出外层 for 循环。case 后没有 break 会"穿透"（fall-through）到下一个 case。
逐步跟踪（for 循环 i = 1 到 4）：

i=1：i%3=1 → 进入 case 1，sum += 1（sum=1）。case 1 后没有 break，穿透到 case 2，sum += 2（sum=3）。case 2 有 break，退出 switch。
i=2：i%3=2 → 进入 case 2，sum += 2（sum=5）。break 退出 switch。
i=3：i%3=0 → 进入 case 0，sum += 10（sum=15）。break 退出 switch。
i=4：i%3=1 → 进入 case 1，sum += 1（sum=16）。穿透到 case 2，sum += 2（sum=18）。break 退出 switch。

for 循环正常结束（i=5 时不满足 i<=4），输出 sum = 18。
易错点：①以为 case 1 的 break 只是漏写了、行为和有 break 一样；②以为 break 会退出 for 循环（实际只退出 switch）；③漏算 i=4 的穿透。
第10题 A（输出9）：#16 continue位置影响
核心考点：在 while 循环中，continue 和 i++ 的相对位置决定了程序是否会死循环。
逐步跟踪：
- i=0: 0%2==0 → i++使i=1, continue（跳过 n+=i）
- i=1: 1%2!=0 → n+=1(n=1), i++(i=2)
- i=2: 2%2==0 → i++使i=3, continue
- i=3: 3%2!=0 → n+=3(n=4), i++(i=4)
- i=4: 4%2==0 → i++使i=5, continue
- i=5: 5%2!=0 → n+=5(n=9), i++(i=6)
- i=6: 不满足 i<6，退出循环
输出 9（= 1 + 3 + 5，即 0~5 中所有奇数之和）。

对比：如果 i++ 放在 continue 之后（如下代码），会发生死循环： 
int n = 0, i = 0;
while (i < 6) {
    if (i % 2 == 0) continue;  // i 永远为 0，死循环！
    n += i;
    i++;
}
因为 i=0 时进入 continue，跳过了 i++，i 永远是 0。
易错点：选 D（死循环），把两种情况搞混了。
第11题 B（不是死循环）：#17 死循环判定
逐个分析：
- A. 死循环：for(int i=10; i>0; i++)。i 从 10 开始递增，永远大于 0（忽略溢出的话），死循环。
- B. 不是死循环 ✓：int x=1; while(x!=10) { x+=3; }。x 的变化：1 → 4 → 7 → 10。当 x=10 时条件 x!=10 为 false，循环正常退出。这是因为步长 3 恰好能到达 10（1+3×3=10）。
- C. 死循环：int n=5; while(n>0) cout<。循环体只输出 n，不修改 n 的值，n 永远是 5，条件永远为 true。
- D. 死循环：for(;;) cout<<1;。for(;;) 等价于 while(true)，无条件无限循环。
易错点：对 B，很多同学会以为"步长不是 1 就到不了 10"，但 1+3+3+3=10 恰好能到。如果改成 x += 2（步长2），则 x 变化为 1→3→5→7→9→11→...，永远不等于 10，就是死循环了（这正是判断题 T9 考的内容）。
第12题 C：#19 宽度+补零+左对齐
分析两个格式说明符：
第一个 %05d（宽度5，0 标志补零，默认右对齐）：
- 42 占 2 位，宽度要求 5 位，左边补 3 个零：00042。
第二个 %-5d（宽度5，- 标志左对齐）：
- 42 占 2 位，左对齐后右边补 3 个空格：42___（_ 表示空格）。
最终输出：[00042][42___]。

printf 格式标志规则总结：
- %5d：宽度 5，右对齐，空格填充 → ___42
- %05d：宽度 5，右对齐，零填充 → 00042
- %-5d：宽度 5，左对齐，空格填充 → 42___
- %-05d：宽度 5，左对齐，0 标志被 - 覆盖（忽略） → 42___
易错点：①把 0 标志的零填充方向搞反（选 A，以为右边补零）；②把 - 标志的对齐方式搞反（选 D，两个都右对齐）。
第13题 A（输出 {%d}=5）：#20 %%和{%d}混合
逐字符解析 printf 格式串 "{%%d}=%d"：
- {：普通字符，原样输出 {。
- %%：转义序列，输出一个 % 字符。注意 %% 是一个完整的转义单元，消耗了两个 %。
- d：此时前面的 %% 已经处理完毕，这个 d 是普通字符，原样输出 d。
- }：普通字符，原样输出 }。
- =：普通字符，原样输出 =。
- %d：格式说明符，用参数 5 替换，输出 5。
最终输出：{%d}=5。
易错点：
- 选 B：以为 %%d 是一个整体的格式符，等同于 %d，把 5 填入其中。实际上 %% 和 d 是分开解析的。
- 选 C：以为 %% 原样输出 %%（实际只输出一个 %）。
- 选 D：以为 %% 消耗了后面的 d。
第14题 C（输出15）：#21 变量交换#24 逗号表达式
第一部分：加减法交换 a 和 b
初始：a=5, b=8。
- a = a + b = 5 + 8 = 13
- b = a - b = 13 - 8 = 5（b 得到了 a 的原始值）
- a = a - b = 13 - 5 = 8（a 得到了 b 的原始值）
交换完成：a=8, b=5。

第二部分：逗号表达式
int c = (a++, b++, a + b);
逗号表达式从左到右依次求值，整个表达式的值是最后一个子表达式的值：
- a++：后置自增，a 的值从 8 变为 9（表达式的值是 8，但 a 已经变了）。
- b++：后置自增，b 的值从 5 变为 6。
- a + b：此时 a=9, b=6，结果为 15。
- c = 15。输出 15。
注意：虽然 a++ 的"表达式值"是 8（旧值），但作为逗号表达式的非最后一个子表达式，这个值被丢弃了。重要的是 a 的副作用（a 变成了 9），这会影响后续的 a+b。
易错点：①把交换算错；②以为 a++ 的值 8 会被使用（实际被逗号表达式丢弃）；③以为逗号表达式输出所有值。
第15题 A（输出 Y6）：#22 完全数#23 GCD辗转相除
第一部分：完全数判断（n=6）
完全数（Perfect Number）是指等于其所有真因子（不包括自身）之和的正整数。
遍历 i = 1 到 5，检查哪些是 6 的因子：
- i=1: 6%1==0 ✓, sum=1
- i=2: 6%2==0 ✓, sum=3
- i=3: 6%3==0 ✓, sum=6
- i=4: 6%4==2 ✗
- i=5: 6%5==1 ✗
sum = 1+2+3 = 6 = n，所以 6 是完全数，输出 "Y"。

第二部分：辗转相除法（GCD(48, 18)）
辗转相除法（欧几里得算法）反复用 a%b 的余数替换，直到余数为 0：
- a=48, b=18: r = 48%18 = 12 → a=18, b=12
- a=18, b=12: r = 18%12 = 6 → a=12, b=6
- a=12, b=6: r = 12%6 = 0 → a=6, b=0
- b=0, 退出循环。GCD = a = 6。

sum==n 成立，执行 cout << "Y" << a，输出 Y6。
易错点：①以为完全数不包含 1 作为因子，漏加 1，得 sum=5≠6（选 B）；②GCD 算法搞混 a 和 b 的更新顺序，以为最终结果是 12 或 48（选 C/D）。

第1题 ✔ (正确)：#25 五大部件
计算机的五大部件（冯·诺依曼体系结构）：
1. 运算器（ALU）— 执行算术和逻辑运算
2. 控制器（CU）— 控制指令的执行顺序
3. 存储器（Memory）— 存储程序和数据
4. 输入设备（Input）— 键盘、鼠标等
5. 输出设备（Output）— 显示器、打印机等
其中运算器 + 控制器 = CPU（中央处理器）。题目说法完全正确。
易错点：把 CPU 当成独立的第六个部件，或把硬盘误归为输入设备（硬盘是存储器的一种）。
第2题 ✘ (错误)：#26 二进制
十进制 13 转二进制：
- 13 ÷ 2 = 6 余 1
- 6 ÷ 2 = 3 余 0
- 3 ÷ 2 = 1 余 1
- 1 ÷ 2 = 0 余 1
从下往上读：1101。
验证：1×8 + 1×4 + 0×2 + 1×1 = 8 + 4 + 0 + 1 = 13 ✓
题目说是 1011，验证：1×8 + 0×4 + 1×2 + 1×1 = 8 + 0 + 2 + 1 = 11 ≠ 13。
1011 是 11 的二进制，不是 13 的。正确答案是 1101。
易错点：除 2 取余后从上往下读（应该从下往上读），或把 1101 和 1011 搞混。
第3题 ✔ (正确)：#27 冯·诺依曼
冯·诺依曼体系结构的核心思想确实是"存储程序"（Stored Program Concept）：
- 将程序指令和数据以同等地位存储在同一个内存中。
- 计算机按照存储的程序指令自动顺序执行，无需人工干预每一步。
这是现代计算机的基本工作原理。在此之前的计算机（如 ENIAC 早期）需要通过重新接线来"编程"。
易错点：把冯·诺依曼和图灵的贡献搞混。图灵提出了图灵机（理论计算模型），冯·诺依曼提出了存储程序的计算机设计思想。
第4题 ✔ (正确)：#28 URL结构
URL https://www.example.com:8080/path?q=1 的完整解析：
- https — 协议（Protocol），表示使用 HTTPS 安全传输协议。
- www.example.com — 域名（Host），标识服务器地址。
- 8080 — 端口号（Port），指定服务器上的具体服务端口。HTTPS 默认端口是 443，这里用了非默认端口 8080。
- /path — 路径（Path），指定服务器上的资源位置。
- q=1 — 查询参数（Query String），? 后面的键值对，用于传递参数。
题目的描述完全正确。
易错点：把端口号和协议搞混，或不知道 ? 后面是查询参数。
第5题 ✘ (错误)：#29 文件类型
题目把 .obj 和 .exe 的说明搞反了。正确对应：
- .cpp — 源代码文件（Source Code），程序员编写的 C++ 代码。 ✓
- .obj — 目标文件（Object File），编译器将源代码编译后生成的中间文件，还不能直接运行。
- .exe — 可执行文件（Executable），链接器将目标文件链接后生成的最终可运行程序。
编译流程：.cpp → (编译) → .obj → (链接) → .exe
易错点：把"目标文件"和"可执行文件"搞混。记忆技巧：obj = object = 目标（中间产物），exe = executable = 可执行（最终产物）。
第6题 ✘ (错误)：#30 编译型语言
C++ 是编译型语言，不是解释型语言。
- 编译型语言（如 C、C++、Go）：源代码需要先由编译器（Compiler）全部翻译成机器码（可执行文件），然后才能运行。运行速度快。
- 解释型语言（如 Python、JavaScript）：源代码由解释器（Interpreter）逐行翻译并执行，不需要提前编译。开发灵活但运行较慢。
题目说"C++ 和 Python 都是解释型语言"是错误的。C++ 是编译型，Python 是解释型。
易错点：把"编译"和"解释"的概念搞混，或者不确定 C++ 属于哪一类。
第7题 ✘ (错误)：#13 转换位置
(int)(3.5 + 4.5) 和 int(3.5) + int(4.5) 的结果不同：
- (int)(3.5 + 4.5)：先加 3.5+4.5=8.0，再转为 int → 8。
- int(3.5) + int(4.5)：先分别转为 int → 3 和 4，再加 → 7。
8 ≠ 7，所以结果不同。
本质原因：类型转换（截断）操作不满足分配律。int(a) + int(b) ≤ int(a+b)（当 a、b 都是正数时）。因为每次截断都可能丢失"不到1"的小数部分，两次截断最多丢失接近 2，而一次截断最多丢失接近 1。
易错点：直觉上觉得"加法前后转换结果一样"，忽略了截断的非线性性。这道题和单选第 2 题相呼应。
第8题 ✘ (错误)：#11 bool算术
在 C++ 中，true 参与算术运算时提升为 int 值 1，false 提升为 0。
- true * 10 = 1 × 10 = 10
- false * 5 = 0 × 5 = 0
- 10 + 0 = 10
结果是 10，不是 15。题目说是 15，错误。
易错点：以为 true 和 false 在算术中都等于 1（或都等于自身的某个"值"），把 false * 5 = 5。记住：false = 0，任何数乘以 0 都是 0。
第9题 ✘ (错误，这是死循环)：#17 死循环判定
int i = 1; while(i != 10) { i += 2; }
i 的变化序列：1 → 3 → 5 → 7 → 9 → 11 → 13 → ...
i 始终是奇数（1 是奇数，每次加 2 仍是奇数），而 10 是偶数。
因此 i != 10 永远为 true，这是一个死循环！
题目说"会正常终止"，错误。
对比单选 Q11 的选项 B：int x=1; while(x!=10) { x+=3; }，x 变化为 1→4→7→10，恰好能到达 10，所以不是死循环。
关键区别：步长为 3 时，1+3×3=10 能到达；步长为 2 时，奇数加偶数个 2 仍是奇数，永远到不了偶数 10。
判断方法：初始值和目标值的差（10-1=9）是否能被步长整除。9÷3=3（整除，能到达），9÷2=4.5（不整除，不能到达）。
第10题 ✘ (错误，x=0 不是 1)：#24 逗号表达式#4 取余
int x = (7 % 3, 8 % 3, 9 % 3);
这是一个逗号表达式，从左到右依次求值，但只取最后一个子表达式的值赋给 x：
- 7 % 3 = 1（计算了，但值被丢弃）
- 8 % 3 = 2（计算了，但值被丢弃）
- 9 % 3 = 0（这是最后一个，值保留）
x = 0，不是 1。题目说 x=1，错误。
易错点：以为逗号表达式取第一个值（选了 7%3=1），或以为逗号表达式会把所有值相加（1+2+0=3）。
逗号表达式核心规则：(expr1, expr2, ..., exprN) 的值等于 exprN 的值。前面的表达式都会被求值（可能有副作用），但它们的值被丢弃。
